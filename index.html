Event Loop: An event loop is something that pulls out of a row and puts it on top of the function execution stack whenever the function stack is empty. It is already known exactly how many times a part of a JavaScript program will repeat or execute in the program, but in that case JavaScript loop provides very convenient results. Loops are used in JavaScript when a task needs to be repeated. JavaScript has a runtime model based on an event loop, which is responsible for running code, collecting and processing events, and running sub-tasks in a row. Event Loop is a process that waits for the call stack to clear before pushing the callback from the task queue to the call stack. Once the stack is clear, trigger the event loop and check the task queue for available callbacks. This means that the main thread where the JavaScript code is executed runs one line at a time and there is no possibility to run the code in parallel.

A unique page session is created when a document is loaded in the Browser tab. Page sessions are only valid for one tab at a time Pages are only saved when tabs or browsers are open; They do not survive after the page has been reloaded and restored. A new session is created each time a tab or window is opened; This is different from session cookies. Each tab / window opened with the same URL creates its own session storage. When you duplicate a tab, the session storage from the main tab is copied to the Duplicate tab. Closing a window / tab ends the session and clears the session storage objects.
